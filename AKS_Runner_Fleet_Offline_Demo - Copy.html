Mixed Runner Fleet — Offline Demo & Reference Guide

Eliminating AWS Egress Costs with Smart Cached Runners

Saurabh Talwar  |  Microsoft

Welcome everyone. This presentation covers our complete AKS runner fleet strategy for eliminating the double-cost problem of AWS data egress + GitHub runner charges.

The Problem: You're Paying Twice

AWS Data Egress Fees

Every CI run copies 5–8 GB from S3/RDS. AWS charges $0.09/GB egress. That's $0.45–$0.72 per run just for data movement.

GitHub Runner Minutes

GitHub-hosted runners charge $0.008/min (Linux). A 15-min job = $0.12/run. No persistent storage means full re-copy every time.

Combined: $0.84/run × 20 runs/month × 50 workflows = $840/month = $10,080/year
Start here to establish the pain. Ask the customer: 'How many workflows do you run daily? How large are your datasets?' Let THEM do the multiplication. The number is always bigger than they expect.

GitHub-Hosted Runners Are Ephemeral

Run 1: AWS S3 ──[8 GB]──> Runner Pod (destroyed) Run 2: AWS S3 ──[8 GB]──> NEW Runner Pod (destroyed) Run 3: AWS S3 ──[8 GB]──> NEW Runner Pod (destroyed)
CALLOUT: Same data. Copied 3 times. Paid 3 times.
Emphasize that ephemeral runners are GREAT for security but TERRIBLE for data-heavy workloads. This isn't a GitHub flaw — it's a design choice that doesn't fit THIS use case.

Q&A — Cost & Pricing

What's the AKS infrastructure cost?
~$280/mo for 2-node D4s_v5 cluster. Scale-to-zero runners mean you only pay node costs during active jobs.
How does this compare to GitHub Larger Runners?
Larger Runners solve compute but NOT egress. You still copy full datasets each run.
What about AKS operational overhead?
2-4 hrs/month. ARC handles runner lifecycle. Azure handles node health and upgrades.
Is there a minimum commitment?
No. Pay-as-you-go. Tear down the cluster anytime. Zero lock-in.
What about Azure Files storage costs?
Standard tier: ~$0.06/GB/mo. 10 GB cache = $0.60/mo.
ROI timeline?
Break-even at Run #2 per workflow. Most see full ROI within the first week.
Can we start small?
Absolutely. Migrate one data-heavy workflow first. Expand based on results.
Hidden costs?
AKS control plane is free. Node VMs + storage + egress (minimal) are the only costs.
Refer to the full Q&A Preparation PDF for detailed responses. Key objection to prepare for: 'What about AKS operational costs?' Answer: 2-4 hrs/month, mostly automated.

Q&A — Technical Depth

How does delta sync work for S3?
ETags + Last-Modified headers. Only objects changed since last sync are fetched.
What about RDS changes?
Timestamp-based queries or CDC (Change Data Capture). Only new/modified rows pulled.
PVC runs out of space?
LRU pruning evicts oldest cached data. Dynamic PVC expansion also supported.
Multiple workflows share cache?
ReadWriteMany access mode. File locking prevents corruption.
AKS cluster goes down?
ARC retries on healthy nodes. Azure Files survives pod/node failures.
Can this work on EKS?
Yes — ARC is Kubernetes-native. Works on any conformant K8s cluster.
These are the questions the senior engineers will ask. Be ready with specifics. If you don't know an answer, say 'Great question — let me get back to you with the exact details' and follow up.

Q&A — Security & Compliance

Where does data go?
YOUR Azure tenant only. Data never traverses GitHub infrastructure.
HIPAA / SOC2 compliant?
AKS is certified for HIPAA, SOC 1/2/3, ISO 27001, FedRAMP, and more.
Does GitHub see our data?
No. GitHub only sees orchestration metadata (job status, timing). Not your actual data.
Network isolation?
Azure CNI + Network Policies + NSGs. Private API server option available.
Runner image vulnerabilities?
Microsoft Defender for Containers scans images. Automated patching via AKS.
Audit trail?
GitHub audit log + AKS diagnostic logs + Azure Activity Log. Full end-to-end.
This is for the security/compliance person. Lead with 'your data never leaves your Azure tenant.' That's the single most important security message.

Q&A — Migration & Vendor Lock-in

How long to production?
POC: 1–2 days. Production with monitoring and hardening: 2–3 weeks.
Incremental migration?
Yes. Change the runs-on label one workflow at a time. No big-bang required.
Rollback plan?
Change runs-on back to ubuntu-latest. Done. One-line rollback.
Locked into Azure?
No. ARC runs on any Kubernetes cluster — EKS, GKE, on-prem, or edge.
Why not AWS CodeBuild?
Ties you to AWS CI/CD. GitHub is where your code lives. Keep CI/CD where the source is.
Migration fear is real. The rollback story is your strongest card: 'change one line back, you're on GitHub-hosted again.' Zero lock-in.
