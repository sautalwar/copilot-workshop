🎤 Speaker Notes
Talk Track — Your Three Priorities (3 min)
This slide frames the three big questions your leadership is asking.

FIRST — Migration. Moving from GHES to GHEC gets you GitHub's managed SaaS with multi-region redundancy, automatic security updates, and the full Copilot experience. GEI handles the heavy lifting: repos, PRs, issues, teams, branch protection rules. The things that need manual work — webhooks, secrets, deploy keys — we'll plan for.

SECOND — Resilience. We're not just moving to the cloud and hoping for the best. Our 4-layer architecture ensures that if ANY layer fails, another catches you. GHEC goes down? GHES keeps running. Azure region fails? GRS replicates to the paired region. Data gets corrupted? Git mirror backups + GHES snapshots give point-in-time recovery.

THIRD — Backup. Defense-in-depth: git mirrors for repo history, API exports for metadata, GHES snapshots for full instance state, and Azure GRS for geo-redundant durability. Together: near-zero RPO, sub-30-minute RTO.

The key message: migration is the beginning, not the end. Resilience and backup ensure your investment is protected.

🎤 Speaker Notes
Talk Track — Architecture Overview (5 min)
Layer A — GHEC: Primary SaaS. Multi-region, 99.9% SLA, built-in security.
Layer B — GHES + HA Replica: Self-hosted, active/passive failover, async replication. Key: HA protects hardware failure, NOT corruption.
Layer C — Azure GRS/GZRS: 16 nines durability. Stores mirrors, metadata, snapshots, LFS. Lifecycle: Hot → Cool → Archive saves 80%.
Layer D — Local/Air-Gapped: Fully offline GHES + encrypted storage. Complete cloud independence.
Together = near-zero RPO, sub-30-minute RTO.

GHES → GHEC Migration & Backup Strategy

Migration via GitHub Enterprise Importer (GEI)
  • 1. Plan — Inventory repos, teams, dependencies
  • 2. Trial — Run trial migrations to validate
  • 3. Migrate — Execute production migration (pause work!)
  • 4. Validate — Verify data fidelity, review logs
  • 5. Cutover — Redirect users, re-enable integrations
⚠️ GEI does NOT support incremental migration
✅ Auto-migrates:
Git source • PRs with reviews • Issues • Wikis • Releases • Teams & permissions • Branch protection → Rulesets
🔧 Manual work:
Webhooks • Actions secrets • Deploy keys • GitHub Apps • Packages • Repos > 40 GB
4-Layer Backup & DR Strategy
Layer 1: Git Mirror
Daily/hourly git clone --mirror — full repo history
Layer 2: Metadata API
Weekly REST/GraphQL export — issues, PRs, releases
Layer 3: GHES Snapshots
Incremental via backup-utils — zero downtime
Layer 4: Azure GRS/GZRS
Geo-redundant cross-region — 16 nines durability
Outage Playbooks:
A. GitHub.com outage → Failover to GHES
B. Azure region outage → GRS failover to paired region
C. Data corruption → Restore from mirror + metadata re-import
🎤 Speaker Notes
Talk Track — Migration & Backup (5 min)
GEI migration: Plan → Trial → Migrate → Validate → Cutover. Critical: NO incremental migration — pause all work during production migration.
Auto-migrates: Git source, PRs, issues, wikis, releases, teams, branch protection.
Manual: Webhooks, secrets, deploy keys, Apps, Packages.
4-Layer Backup: Git mirror (hourly), Metadata API (weekly), GHES snapshots (incremental, zero downtime), Azure GRS (geo-redundant).
Outage playbooks: GitHub.com down → GHES, Azure down → paired region failover, Corruption → mirror restore + metadata re-import.
