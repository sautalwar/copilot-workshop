Presenter: Saurabh Talwar, Microsoft | Duration: ~3.5 hours

📋Workshop Overview

We're excited to invite you to our hands-on GitHub Copilot workshop. This is not a presentation — it's a working garage session where you'll get your hands on an AI coding assistant and learn how it can help you write code faster.

What to Expect

  • Format: Hands-on, interactive — bring your laptop and be ready to code along with the group
  • Duration: Approximately 3.5 hours
  • Skill level: All levels welcome — no prior AI coding experience needed

What You'll Learn

  • Setting up and configuring GitHub Copilot in your IDE
  • AI-assisted coding — code completions, chat interactions, and inline suggestions
  • Choosing the right model for your task
  • MCP (Model Context Protocol) integrations — connecting Copilot to external tools and data
  • Security and governance — how Copilot fits into enterprise policies
  • Real-world patterns for using AI to accelerate development

💻Hardware Requirements

What You'll Need

  • Laptop: Windows, macOS, or Linux — any recent model
  • RAM: 8GB or more recommended (4GB minimum, but you'll feel the difference)
  • Disk space: ~5GB free (for JDK, Maven dependencies, Docker images)
  • Internet: Stable, high-speed connection (required for Copilot, GitHub, model marketplace)
  • Display: Second monitor helpful but not required (dual monitors make it easier to follow along)
💡 Pro Tip: If you're running on 4GB RAM, close unnecessary applications before the workshop. You may also want to skip the Docker/SQL Server demos and focus on the core IDE setup.

1. JDK 17 (Java Development Kit)

Why: Our workshop app requires Java 17. You can't skip this — it's the foundation for everything else.

Download: https://adoptium.net/

Choose: Temurin 17 LTS (Long Term Support) for your operating system

Windows

  1. Click the download link for Windows (`.msi` installer)
  2. Run the installer
  3. IMPORTANT: During installation, check these options:
    • ☑ Set JAVA_HOME environment variable
    • ☑ Add to PATH
  4. Click Next and finish
  5. Restart your computer

macOS

  1. Option A (Homebrew — easier): 
    brew install temurin@17
  2. Option B (Manual): Download the `.pkg` installer and run it

Linux (Ubuntu/Debian)

  1. sudo apt-get update
sudo apt-get install -y temurin-17-jdk

Verify Installation

Open a new terminal/command prompt and run:

java --version

Expected output:

openjdk 17.0.x 2024-xx-xx
OpenJDK Runtime Environment Temurin-17.0.x+x
✅ Success: You see "openjdk 17" in the output
⚠️ Problem? See the "Troubleshooting" section below

2. Apache Maven 3.9+

Why: Maven is the build tool that compiles our Java app and manages dependencies.

Download: https://maven.apache.org/download.cgi

Choose: Binary zip archive (e.g., apache-maven-3.9.x-bin.zip)

Windows

  1. Download the `.zip` file
  2. Extract it to C:\tools\maven (or anywhere convenient)
  3. Add to your system PATH:
    • Right-click "This PC" / "My Computer" → Properties
    • Click "Advanced system settings"
    • Click "Environment Variables"
    • Under "System variables," click "Path" → Edit
    • Click "New" and add: C:\tools\maven\apache-maven-3.9.x\bin (replace `x` with your version)
    • Click OK, OK, OK
    • Restart your computer

macOS / Linux

  1. Extract to /opt/maven or ~/tools/maven
  2. Add to your shell profile (~/.bashrc, ~/.zshrc, etc.): 
    export PATH=/opt/maven/apache-maven-3.9.x/bin:$PATH
  3. Reload: source ~/.bashrc

Verify Installation

Open a new terminal and run:

mvn --version

Expected output:

Apache Maven 3.9.x ...
✅ Success: You see "Apache Maven 3.9" in the output

📦Repository Setup & Quick Test

Let's make sure your environment is working correctly by cloning the workshop repository and running the app locally.

Step 1: Clone the Repository

Open a terminal in a folder where you want to store the project, then run:

git clone https://github.com/sautalwar/outfront-copilot-workshop.git
cd outfront-copilot-workshop

Step 2: Build and Start the App

mvn spring-boot:run

What to expect:

  • Maven will download dependencies (may take 1–2 minutes on first run)
  • The application will start and print: Started Application
  • The app is now running on http://localhost:8080

Step 3: Verify It Works

Test 1: API Endpoint

Open your browser and go to:

http://localhost:8080/api/orders

You should see JSON data with sample orders:

{
  "id": 1,
  "customerName": "Acme Corp",
  "orderStatus": "PENDING",
  ...
}

Test 2: API Documentation

Open your browser and go to:

http://localhost:8080/swagger-ui.html

You should see interactive API documentation with all endpoints listed.

Step 4: Stop the App

In the terminal where the app is running, press:

Ctrl+C
✅ Success: The app starts, you see JSON data, and it stops cleanly with Ctrl+C.
💡 Database Note: The app uses H2, an in-memory database that auto-loads sample data on startup. No external database server needed.

✨Copilot Verification — Critical Step

This is the most important check. If Copilot isn't generating suggestions, the workshop will be hard to follow along with.

How to Test

  1. Open the workshop project in VS Code
  2. Navigate to any .java file, for example: 
    src/main/java/com/outfront/workshop/controller/OrderController.java
  3. Click at the end of any line inside a method body
  4. Type a comment like: 
    // method to calculate order total including tax
  5. Pause for 2–3 seconds and look for ghost text — grayed-out suggestions that appear below your comment
  6. If you see suggestions:
    • Press Tab to accept a suggestion, or
    • Press Esc to dismiss it
✅ Copilot is working: You see grayed-out code suggestions appear below your comment.
⚠️ No suggestions appearing? Check the troubleshooting section below. This is usually a sign-in issue.

⚪SQL Server — You Do NOT Need to Install This

🎯 KEY POINT: You do NOT need to install SQL Server. Our default setup uses H2, an in-memory database that starts automatically with zero configuration.

Why We Mention SQL Server

SQL Server exists only for the optional production database profile demo. Here's the breakdown:

  • Default (what you'll use): H2 in-memory database. Starts instantly, auto-seeded with data. Zero setup. ✅
  • Optional production profile: SQL Server 2022 (via Docker Compose). For advanced demos only.

If You Want to See the Production Database (Not Required)

We provide Docker Compose to spin up SQL Server:

docker compose up -d sqlserver

Then start the app with the SQL Server profile:

mvn spring-boot:run -Dspring-boot.run.profiles=sqlserver

But: This requires Docker Desktop, which is optional. Skip this if you don't have Docker or you're short on disk space.

🔧Troubleshooting FAQ

Stuck? Here are solutions to the most common issues.

❌ "java" command not found or not recognized
Cause: JAVA_HOME environment variable is not set, or it's not in your system PATH.
Fix:
  • Verify JDK is installed: Look for the JDK folder (e.g., C:\Program Files\Eclipse Adoptium\jdk-17.0.x on Windows)
  • Set JAVA_HOME:
    • Windows: Go to System Properties → Environment Variables → New → JAVA_HOME = path to JDK folder
    • Mac/Linux: Add to ~/.bashrc or ~/.zshrc: export JAVA_HOME=/path/to/jdk
  • Restart your terminal
  • Try java --version again
❌ "mvn" command not found or not recognized
Cause: Maven is not in your system PATH.
Fix:
  • Verify Maven is extracted: Look for the folder apache-maven-3.9.x/bin
  • Add Maven bin to PATH:
    • Windows: System Properties → Environment Variables → Edit "Path" → Add C:\tools\maven\apache-maven-3.9.x\bin (adjust path)
    • Mac/Linux: Add to ~/.bashrc or ~/.zshrc: export PATH=/path/to/maven/bin:$PATH
  • Very important: Close your terminal completely and open a NEW one
  • Try mvn --version again
❌ Copilot extension shows "Disabled" or suggestions aren't appearing
Causes: Sign-in issue, extension not enabled, or Copilot is disabled in settings.
Fix:
  1. Check sign-in status: Click the Copilot icon in the status bar (bottom-right) — it should show "Signed in"
  2. If signed out, click "Sign in with GitHub" and use your OutFront EMU account
  3. Make sure both extensions are installed:
    • Ctrl+Shift+X → Search "GitHub Copilot" → should say "Disable" (meaning it's enabled)
    • Ctrl+Shift+X → Search "GitHub Copilot Chat" → should say "Disable" (meaning it's enabled)
  4. Restart VS Code entirely (close it completely, then reopen)
  5. Try typing a comment in a .java file again and wait 2–3 seconds for suggestions

Still nothing? Check your OutFront GitHub account has Copilot license assigned. Contact your IT admin if unsure.

❌ "Invalid target release: 17" when running mvn build
Cause: Your JDK is older than version 17.
Fix:
  • Run java --version — note the version number
  • If it's less than 17 (e.g., 11, 8), you need to install JDK 17
  • Download from https://adoptium.net/ and install Temurin 17
  • After installation, restart your terminal and verify: java --version
❌ "JAVA_HOME is not set" error
Cause: Maven can't find JAVA_HOME environment variable.
Fix:
  • Windows:
    1. Right-click "This PC" → Properties
    2. Advanced system settings → Environment Variables
    3. New system variable: JAVA_HOME = C:\Program Files\Eclipse Adoptium\jdk-17.0.x.x-hotspot (adjust path)
    4. Restart computer
  • Mac/Linux: Add to ~/.bashrc or ~/.zshrc: 
    export JAVA_HOME=$(/usr/libexec/java_home -v 17)
    Then: source ~/.bashrc
❌ Maven build fails with dependency download errors
Cause: Network issue or corporate proxy blocking Maven repository.
Fix:
  • Check your internet connection
  • If behind a corporate proxy, configure Maven's settings.xml with proxy details. Ask your IT admin for proxy server, port, username, password.
  • Try the build again: mvn clean verify
  • If still stuck, try: mvn -U clean verify (forces update of dependencies)
❌ Port 8080 already in use when starting the app
Cause: Another application is using port 8080.
Fix:
  • Windows: netstat -ano | findstr :8080 → find PID → taskkill /PID {pid}
  • Mac/Linux: lsof -i :8080 → find process → kill -9 {pid}
  • Or change the port: mvn spring-boot:run -Dspring-boot.run.arguments="--server.port=8081"
❌ SSO / GitHub Enterprise sign-in issues
Cause: Browser redirect loop, wrong account, or SSO not properly configured.
Fix:
  • Make sure you're signing in with your OutFront GitHub Enterprise account (not a personal account)
  • Clear browser cookies for github.com and try again
  • Try signing in via the browser directly first: Go to your GitHub Enterprise URL, sign in manually
  • Then try VS Code: Press Ctrl+Shift+P → "GitHub Copilot: Sign In"
  • If still stuck, contact your IT/GitHub admin — SSO issues may require configuration on the enterprise side
❌ Git clone fails with permission denied
Cause: Not authenticated with GitHub or SSH key not configured.
Fix:
  • Use HTTPS instead of SSH: git clone https://github.com/sautalwar/outfront-copilot-workshop.git
  • You'll be prompted for username/password (or token) — use your OutFront GitHub credentials
  • Or configure Git: git config --global user.email "your@email.com"
❌ App starts but returns 404 when accessing API endpoints
Cause: App started on a different port or endpoints are incorrect.
Fix:
  • Check the app startup log — it prints the port (usually 8080)
  • Verify you're accessing the right URL: http://localhost:8080/api/orders (not /orders alone)
  • Check the API docs: http://localhost:8080/swagger-ui.html — this shows all available endpoints
