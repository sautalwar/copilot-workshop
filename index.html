[TIME 12:30-17:00 | 4.5 min] This is our ACTUAL repository structure. Let me explain each piece. INSTRUCTIONS: Think of these as the constitution. copilot-instructions.md is 45KB of rules that Copilot ALWAYS follows — coding style, naming conventions, commit format, testing frameworks, security rules. Path-specific instructions add extra rules when editing matching files. Instructions never change the model or tools — they add behavioral guardrails. PROMPT FILES: Think of these as saved recipes. You invoke them by name to do a specific task. They CAN specify a model and tools. They are NOT always active — only on demand. CHAT MODES: Think of these as entirely different AI personalities. When you switch to a chat mode, it REPLACES default Copilot with a custom system prompt, model, and tools. PRECEDENCE: Instructions are always on (base layer). Prompts add on top for one task. Chat modes take over entirely for a session. CONTENT EXCLUSION: Configured at org level in GitHub Settings, not in the repo. Tells Copilot to completely ignore certain files. Use cases: .env files, proprietary algorithms, PII data, regulated financial code. CONTENT INCLUSION: The escape hatch. Override exclusion for specific paths when teams need AI help with certain files inside excluded directories.

Content Governance

Control what the AI can see — and what it cannot

TIME 34:00-34:30 | 30 sec | Enterprise governance is where GitHub Copilot provides powerful controls for security and compliance teams.

Content Exclusion and Inclusion — How It Works

Step-by-step: where to configure and what happens

Where to Configure

  1. Go to github.com → Your Organization
  2. Click Settings → Copilot in sidebar
  3. Select Content exclusion
  4. Add repositories and paths

What Happens When Excluded

  • No completions
  • Chat won't use as context
  • Agent won't read/modify
  • Coding Agent won't include
  • Code Review won't analyze

github.com/organizations/contoso/settings/copilot/content_exclusion

Content exclusion

Repository

Paths

TIME 34:30-37:00 | 2.5 min | Let me show you exactly WHERE you configure content exclusion. On GitHub.com, go to your organization page, click Settings, in the left sidebar click Copilot, then Content exclusion. You add repositories and glob patterns like **/.env to block all env files. Once saved, the exclusion takes effect across ALL Copilot features — completions, chat, agent mode, coding agent, code review. The exclusion is enforced server-side — developers cannot override it.
